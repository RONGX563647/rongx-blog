第二章：CSS基础与盒模型 — 网页的皮肤

费曼学习法：如果不能用"俄罗斯套娃"讲清楚盒模型，说明你还没真正理解 CSS。

本章目标：掌握 CSS 选择器、层叠规则，并彻底理解盒模型 —— 这是后面所有布局知识的基石。

🎨 2.1 费曼入门：什么是CSS？

🏠 装修队比喻

上章说了 HTML = 钢筋骨架（决定了有几个房间、门窗在哪）。
CSS = 装修队 负责所有"看起来怎样"的事情：
• 墙壁刷什么颜色（background-color）
• 家具怎么摆（布局 position / flex / grid）
• 灯光的亮度和颜色（color, opacity）
• 房间有多大（width, height）
• 门离墙多远（margin, padding）

一句话总结：CSS 就是选择 HTML 元素 + 定义样式规则的语言。一条 CSS 规则 = 选择器 + 声明块。

/* CSS规则的解剖结构 */
h1 {                         /* ← 选择器：选中所有 h1 */
    color: #333;            /* ← 声明：属性 + 值 + 分号 */
    font-size: 24px;
}

🎯 2.2 CSS选择器：精准选中目标元素

选择器就是 CSS 的"瞄准镜"。选不中目标，样式就不生效。这是初学者最常遇到的问题。

选择器语法选中什么权重
元素选择器p { }所有 <p> 元素1
类选择器.box { }所有 class="box" 的元素10
ID选择器#header { }id="header" 的元素（唯一）100
后代选择器div p { }div 内部的所有 p（任意深度）1+1
子代选择器div > p { }div 直接子元素 p（仅一层）1+1

💡 最常用的三种选择器（记这三个就够80%的场景）

1. .class —— 日常主力，复用性强（推荐）
2. #id —— JS 定位用，CSS 少用（特异性太高，难覆盖）
3. 后代选择器 nav a —— 限定作用域，避免误伤

⚖️ 2.3 CSS的"层叠"：当多条规则冲突时谁赢？

CSS 的 C 是 Cascade（层叠）。当多个样式规则都指向同一个元素时，浏览器按优先级决定谁说了算：

1
!important（核武器，慎用）

权重 ∞，写在属性值后。一旦使用，几乎无法被覆盖。不到万不得已不要用！

2
内联样式 style="..."

直接写在 HTML 标签上，权重 1000。优先级高于任何外部样式表。

3
ID选择器 #id（权重100）

每个ID选择器贡献100分。

4
类/属性/伪类选择器 .class [attr] :hover（权重10）

每个贡献10分。

5
元素/伪元素选择器 div ::before（权重1）

每个贡献1分。

6
同权时：后面的覆盖前面的

相同优先级，谁写在后面谁生效。这也是"层叠"的本意 —— 像瀑布一样从上往下流。

🧩 记忆口诀：权重计分法

ID百位 类十位 标签个位

#nav .item a → 100 + 10 + 1 = 111分

.nav .item → 10 + 10 = 20分

📦 2.4 盒模型（Box Model）—— CSS最核心的概念

在CSS眼中，页面上的每一个HTML元素都是一个"盒子"。 理解了盒模型，你就理解了80% CSS布局问题的本质。

🎁 快递包裹比喻（最经典的费曼类比）

content（内容） = 你买的商品本身 —— width × height 决定的区域
padding（内边距） = 泡沫填充物 —— 内容到盒子边缘的距离
border（边框） = 快递纸盒的厚度 —— 包围内容和内边距
margin（外边距） = 包裹和包裹之间的间距 —— 这个盒子和其他盒子的距离

一个盒子的总宽度 = content + padding-left + padding-right + border-left + border-right
margin 不计入宽度，但它决定了盒子和邻居之间的距离。

📏 margin（外边距）- 橙色虚线区域
📐 border（边框）- 紫色实线
🧽 padding（内边距）- 绿色虚线区域
📦 content（内容区）- 蓝色区域
width × height

✅ box-sizing: border-box —— 你必须知道的救命属性

默认情况下（box-sizing: content-box），width 只算内容区，加上 padding 和 border 后盒子实际宽度会变大。
设置 box-sizing: border-box 后，width 包含 content + padding + border，盒子总宽就是 width。

这就是为什么几乎每个项目都在全局写：
* { box-sizing: border-box; }
甚至我们的这个 PPT 页面也写了这行 —— 因为不用它，你永远算不准盒子到底多宽。 

/* ★ 几乎每个项目的标配 */
*, *::before, *::after {
    box-sizing: border-box;
}

/* 不设置 border-box 的后果： */
.card {
    width: 300px;           /* 你以为盒子总宽 = 300px */
    padding: 20px;          /* 实际总宽 = 300 + 20×2 = 340px ！*/
    border: 2px solid;     /* 加上边框 = 344px ！！*/
}

/* 设置 border-box 后： */
.card {
    box-sizing: border-box;
    width: 300px;           /* 盒子总宽就是 300px，稳！ */
    padding: 20px;
}

🔲 2.5 display 属性：决定盒子的"行为模式"

display 是最重要的 CSS 属性之一，它决定了元素用哪种"盒模型"来排列：

display值行为能设宽高？独占一行？
block块级盒子，独占一行，宽度默认100%✅ 能✅ 是
inline行内盒子，像文字一样排列❌ 不能❌ 否
inline-block行内排列，但能设宽高✅ 能❌ 否
none完全隐藏，不占空间——
flex弹性布局容器（下章重点）✅ 能✅ 是
grid网格布局容器（第5章重点）✅ 能✅ 是

⚠️ inline元素的陷阱

给一个 <span>（默认 inline）设置 width: 200px 或 margin-top: 20px不会生效！
需要改为 display: inline-block 或 display: block。
这是无数初学者踩过的坑。
