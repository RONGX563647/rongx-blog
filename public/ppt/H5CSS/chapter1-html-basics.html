第一章：HTML基础 — 网页的骨架

费曼学习法：如果你不能把 HTML 解释给一个完全不懂编程的人听，说明你还没真正理解它。

本章目标：学完之后，你能用一句话说清楚 HTML 是什么，并且能手写一个结构良好的页面骨架。

🏗️ 1.1 费曼入门：什么是HTML？

🏠 盖房子比喻

HTML = 房子的钢筋骨架
你走进一栋还没装修的毛坯房，看到的是：
• 承重墙（决定房间结构）—— 对应 HTML 的块级元素
• 门框、窗框（开口和通道）—— 对应 HTML 的表单和链接
• 房间标签（"主卧"、"厨房"）—— 对应 HTML 的语义标签

没有骨架，房子会塌。没有 HTML，网页就不存在。
CSS 是装修（决定好不好看），JavaScript 是水电系统（决定能不能动）。

一句话总结：HTML（HyperText Markup Language）是一种标记语言，用标签（tag）来告诉浏览器"这里放什么内容、结构怎么组织"。

📄 1.2 HTML文档的基本结构

每一个HTML页面都是一份"标准文件"，就像一封正式信件有固定的格式：

DOCTYPE

文档类型声明

→
html

根元素

→
head

元信息

→
body

可见内容

<!-- 1. 文档声明：告诉浏览器"我是HTML5页面" -->
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <!-- 字符编码：支持中文等各种字符 -->
    <meta charset="UTF-8">
    <!-- 视口设置：适配移动端 -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <!-- 浏览器标签栏显示的标题 -->
    <title>我的网页</title>
</head>
<body>
    <!-- 所有用户能看到的内容都写在这里 -->
    <h1>你好，世界！</h1>
</body>
</html>

💡 费曼理解法：head vs body

head = 信封上的地址和邮戳（浏览器需要读，但用户看不到）
body = 信封里的信纸内容（用户真正看到的东西）
把 charset、title、meta 放 head；把 h1、p、div 放 body。

🏷️ 1.3 常用HTML标签速查

HTML标签就像工具箱里的各种工具，每样工具干一件事：

标签用途类比类型
<h1>~<h6>标题（1级最大，6级最小）报纸头条 vs 小标题块级
<p>段落文章中的一个自然段块级
<div>通用容器（无语义）快递包装盒块级
<span>行内容器（无语义）荧光笔划的线行内
<a>超链接门牌号→另一个房间行内
<img>图片（自闭合）墙上挂的画行内块
<ul>/<ol>/<li>列表（无序/有序）购物清单块级
<table>表格Excel表格块级
<form>/<input>表单银行填单块级/行内块
<br>换行（自闭合）回车键—
<hr>水平分割线（自闭合）书签分隔线块级

✅ 核心要点：块级 vs 行内

块级元素（div, p, h1）：独占一行，宽度默认100%，可以包住其他元素。
行内元素（span, a, strong）：像流水一样排在行内，宽度由内容决定。
这直接决定了后面 CSS 布局的基础 —— 这是整个布局理论的地基。

🧠 1.4 语义化HTML：让代码"会说话"

早期的 HTML 代码满屏都是 <div>，搜索引擎和屏幕阅读器完全不知道每个 div 是干什么的。语义化就是给每个区域一个"有意义的标签"：

<header>

页眉区域，通常放Logo和导航

<nav>

导航链接区域

<main>

页面主体内容（每页一个）

<article>

独立的文章/内容块

<section>

主题分组区域

<aside>

侧边栏/附加内容

<footer>

页脚，版权信息等

<figure>/<figcaption>

带说明的图片/图表

📦 语义化 = 给盒子贴标签

以前：10个一模一样的棕色纸箱堆在一起，你要一个一个打开才知道里面是什么。
语义化：每个箱子上写着"冬装"、"厨具"、"工具" —— 搜索引擎（快递员）一眼就知道该把哪个箱子送到哪。 

<!-- 语义化页面结构示例 -->
<body>
    <header>
        <nav>...导航链接...</nav>
    </header>
    <main>
        <article>
            <section>第一部分</section>
            <section>第二部分</section>
        </article>
        <aside>侧边栏</aside>
    </main>
    <footer>版权信息</footer>
</body>

📝 1.5 表单：用户和网页对话的通道

没有表单，网页就是一张静态的海报。有了表单，用户才能输入、选择、提交。

<form action="/submit" method="POST">
    <label for="name">姓名：</label>
    <input type="text" id="name" placeholder="请输入姓名">

    <label for="email">邮箱：</label>
    <input type="email" id="email">

    <button type="submit">提交</button>
</form>
input type效果常用属性
text单行文本输入placeholder, maxlength, required
password密码输入（圆点隐藏）minlength, autocomplete
email邮箱（自动校验格式）multiple
number数字（带增减箭头）min, max, step
checkbox多选框checked
radio单选框（同name分组）name, value, checked
file文件上传accept, multiple
date日期选择器min, max

💡 label + for 的重要习惯

始终用 <label for="id"> 关联表单元素。点击 label 文字会自动聚焦到对应输入框 —— 这是用户体验的基本素养，也是无障碍访问（a11y）的基础。

🔧 1.6 HTML属性：标签的"配置项"

属性就是标签的附加说明，格式是 属性名="属性值"：

属性用于说明
id任意元素全局唯一标识，CSS/JS定位用。页面中不能重复！
class任意元素CSS样式分类，可以多个（空格分隔），可以重复
style任意元素内联样式（不推荐大量使用）
hrefa, link链接地址
srcimg, script, iframe资源路径
altimg图片替代文本（SEO和无障碍必须！）
data-*任意元素自定义数据属性，JS读取用
disabledinput, button禁用元素（布尔属性，不需要值）

🧩 记忆口诀：id vs class

id如身份证号 —— 一人一个，绝不重复

class如标签贴 —— 一物多个标签，多物同一标签
