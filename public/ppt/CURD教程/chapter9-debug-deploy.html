第九章：调试与部署 — 让你的代码"活"在互联网上

费曼学习法：代码在本地跑通只完成了50%，让它安全、稳定、快速地在互联网上运行，才是真正的挑战。

本章教你两件事：🔥灭火（调试bug）和🏗️盖房子（部署上线）——从浏览器DevTools到Docker+K8s，从本地开发到全球CDN。

本章目标：能独立把项目部署到云服务器上，并排查90%的线上问题。

// 常见问题排查清单： // ✅ 1. 检查请求URL是否正确 // Network面板 → 点击请求 → Headers → General → Request URL // ✅ 2. 检查请求方法是否正确（GET/POST/PUT/DELETE） // Network面板 → Headers → Request Method // ✅ 3. 检查请求头是否正确 // Network面板 → Headers → Request Headers // 重点关注：Content-Type、Authorization // ✅ 4. 检查请求参数是否正确 // Network面板 → Payload（POST请求）或 Query String Parameters（GET请求） // ✅ 5. 检查响应状态码 // 200: 成功 // 400: 请求参数错误 // 401: 未授权（Token过期或无效） // 403: 权限不足 // 404: 接口不存在 // 500: 服务器内部错误 // ✅ 6. 检查响应数据格式 // Network面板 → Response → 查看返回的JSON数据 // ✅ 7. 复制为cURL命令（用于后端复现问题） // Network面板 → 右键请求 → Copy → Copy as cURL

💡 CI/CD最佳实践

1. 小步快跑：频繁提交代码，每次提交都触发CI

📝 监控场景

📈 1. Spring Boot Actuator健康检查

org.springframework.boot spring-boot-starter-actuator # 2. 配置application.yml management: endpoints: web: exposure: include: health,info,metrics,prometheus endpoint: health: show-details: always show-components: always health: db: enabled: true redis: enabled: true diskspace: enabled: true # 3. 自定义健康指标 @Component public class CustomHealthIndicator implements HealthIndicator { @Override public Health health() { try { // 检查外部服务状态 boolean serviceAvailable = checkExternalService(); if (serviceAvailable) { return Health.up() .withDetail("service", "running") .withDetail("timestamp", Instant.now()) .build(); } else { return Health.down() .withDetail("service", "unavailable") .build(); } } catch (Exception e) { return Health.down(e).build(); } } }

📈 2. 接口性能监控（AOP）

// PerformanceMonitorAspect.java @Aspect @Component @Slf4j public class PerformanceMonitorAspect { private static final long SLOW_THRESHOLD = 1000; // 慢接口阈值：1秒 @Around("@within(org.springframework.web.bind.annotation.RestController)") public Object monitorPerformance(ProceedingJoinPoint point) throws Throwable { long startTime = System.currentTimeMillis(); String methodName = point.getSignature().toShortString(); try { Object result = point.proceed(); long costTime = System.currentTimeMillis() - startTime; if (costTime > SLOW_THRESHOLD) { log.warn("慢接口警告: {} 耗时: {}ms", methodName, costTime); } return result; } catch (Exception e) { long costTime = System.currentTimeMillis() - startTime; log.error("接口异常: {} 耗时: {}ms, 错误: {}", methodName, costTime, e.getMessage()); throw e; } } }

🧠 费曼自测：你能解释给同学听吗？

Q1：Docker到底是什么？（用集装箱比喻）

A：传统部署就像把每件家具单独搬上船——容易磕碰且占空间。Docker就像集装箱——把应用+依赖+环境打包成一个标准箱子，无论在哪艘船（服务器）上都能直接运行。docker-compose就像一列火车，把多个集装箱（前端+后端+数据库）串在一起发车。

Q2：Nginx反向代理为什么要加？

A：就像公司前台——访客（用户）只看到前台小姐（Nginx:80端口），不知道后台有多少人在干活（多个Spring Boot:8080实例）。前台帮你转接、限流、缓存、检查身份证（SSL），保护后台员工的安全。

Q3：K8s和Docker Compose有什么区别？

A：Docker Compose就像手动开车——你坐驾驶位，自己决定去哪、加速、停车。K8s就像自动驾驶——你告诉它目的地（期望状态），它自动决定走哪条路、换多少个轮子（Pod）、如何处理堵车（自动扩容）。3个服务器用Compose够了，30个服务器必须K8s。
