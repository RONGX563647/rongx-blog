第四章：Java后端CRUD开发 — 像流水线一样处理数据

费曼学习法：后端开发本质上就是一条数据流水线。前端传来请求 → Controller接单 → Service加工 → Mapper存库 → 结果原路返回。

本章用最清晰的三层架构，带你理解Spring Boot如何把数据从"毛坯"加工成"精装"。

本章目标：掌握Controller→Service→Mapper的分层思想，任何CRUD接口都是这个模板的变体。

🏭 费曼类比：后端 = 工厂流水线

Controller（接单员） = 前台客服，接待客户、验证订单格式，但不亲自干活
Service（车间工人） = 真正的加工者，调用工具（Mapper）、执行工艺（业务逻辑）
Mapper（仓库管理员） = 只管存取货架，你说"取第3排第5个"，它就去拿；你说"放第7排"，它就放

为什么分三层？因为客服生病了换一个人就好，车间换工艺不需要客服重新培训，货架换数据库品牌不关车间的事。这就是"解耦"——各司其职，互不干扰。

import com.baomidou.mybatisplus.annotation.*;
import lombok.Data;
import java.time.LocalDateTime;

@Data
@TableName("users")
public class User {
  @TableId(type = IdType.AUTO)
  private Long id;

  private String username;
  private String email;
  private String phone;
  private Integer status;

  @TableLogic
  private Integer deleted;

  @TableField(fill = FieldFill.INSERT)
  private LocalDateTime createTime;

  @TableField(fill = FieldFill.INSERT_UPDATE)
  private LocalDateTime updateTime;
}

import lombok.Data;

@Data
public class Result<T> {
  private Integer code;
  private String msg;
  private T data;

  public static <T> Result<T> success() {
    return success(null);
  }

  public static <T> Result<T> success(T data) {
    Result<T> result = new Result<>();
    result.setCode(200);
    result.setMsg("成功");
    result.setData(data);
    return result;
  }

  public static <T> Result<T> error(String msg) {
    return error(500, msg);
  }

  public static <T> Result<T> error(Integer code, String msg) {
    Result<T> result = new Result<>();
    result.setCode(code);
    result.setMsg(msg);
    return result;
  }
}

⚙️ 4.7 Service层（业务逻辑）

Service接口

import com.baomidou.mybatisplus.extension.service.IService;
import com.example.demo.dto.UserDTO;
import com.example.demo.dto.UserQueryDTO;
import com.example.demo.entity.User;
import com.example.demo.common.PageResult;

public interface UserService extends IService<User> {
  PageResult<User> queryPage(UserQueryDTO queryDTO);
  void createUser(UserDTO userDTO);
  void updateUser(Long id, UserDTO userDTO);
  void deleteUser(Long id);
}

Service实现类

import com.baomidou.mybatisplus.core.conditions.query.LambdaQueryWrapper;
import com.baomidou.mybatisplus.extension.plugins.pagination.Page;
import com.baomidou.mybatisplus.extension.service.impl.ServiceImpl;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;
import org.springframework.util.StringUtils;

@Service
public class UserServiceImpl extends ServiceImpl<UserMapper, User> implements UserService {

  @Override
  public PageResult<User> queryPage(UserQueryDTO queryDTO) {
    // 构建分页对象
    Page<User> page = new Page<>(queryDTO.getPageNum(), queryDTO.getPageSize());

    // 构建查询条件
    LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
    wrapper.like(StringUtils.hasText(queryDTO.getUsername()), User::getUsername, queryDTO.getUsername())
        .eq(queryDTO.getStatus() != null, User::getStatus, queryDTO.getStatus())
        .orderByDesc(User::getCreateTime);

    // 执行分页查询
    Page<User> resultPage = this.page(page, wrapper);

    return new PageResult<>(resultPage.getRecords(), resultPage.getTotal());
  }

  @Override
  @Transactional // 事务管理
  public void createUser(UserDTO userDTO) {
    // 检查用户名是否存在
    long count = this.count(new LambdaQueryWrapper<User>()
        .eq(User::getUsername, userDTO.getUsername()));
    if (count > 0) {
      throw new RuntimeException("用户名已存在");
    }

    User user = new User();
    user.setUsername(userDTO.getUsername());
    user.setEmail(userDTO.getEmail());
    user.setPhone(userDTO.getPhone());
    user.setStatus(userDTO.getStatus() != null ? userDTO.getStatus() : 1);

    this.save(user);
  }

  @Override
  @Transactional
  public void updateUser(Long id, UserDTO userDTO) {
    User user = this.getById(id);
    if (user == null) {
      throw new RuntimeException("用户不存在");
    }

    user.setEmail(userDTO.getEmail());
    user.setPhone(userDTO.getPhone());
    user.setStatus(userDTO.getStatus());

    this.updateById(user);
  }

  @Override
  @Transactional
  public void deleteUser(Long id) {
    User user = this.getById(id);
    if (user == null) {
      throw new RuntimeException("用户不存在");
    }

    // 配置了@TableLogic后，这里会执行逻辑删除而非物理删除
    this.removeById(id);
  }
}

🎮 4.8 Controller层（接口层）

import com.example.demo.common.PageResult;
import com.example.demo.common.Result;
import com.example.demo.dto.UserDTO;
import com.example.demo.dto.UserQueryDTO;
import com.example.demo.entity.User;
import com.example.demo.service.UserService;
import org.springframework.validation.annotation.Validated;
import org.springframework.web.bind.annotation.*;
import javax.annotation.Resource;

@RestController
@RequestMapping("/api/users")
public class UserController {

  @Resource
  private UserService userService;

  // 查询用户列表（分页）
  @GetMapping
  public Result<PageResult<User>> list(UserQueryDTO queryDTO) {
    PageResult<User> result = userService.queryPage(queryDTO);
    return Result.success(result);
  }

  // 新增用户
  @PostMapping
  public Result<Void> create(@Validated @RequestBody UserDTO userDTO) {
    userService.createUser(userDTO);
    return Result.success();
  }

  // 更新用户
  @PutMapping("/{id}")
  public Result<Void> update(@PathVariable Long id, @Validated @RequestBody UserDTO userDTO) {
    userService.updateUser(id, userDTO);
    return Result.success();
  }

  // 删除用户
  @DeleteMapping("/{id}")
  public Result<Void> delete(@PathVariable Long id) {
    userService.deleteUser(id);
    return Result.success();
  }

  // 获取用户详情
  @GetMapping("/{id}")
  public Result<User> detail(@PathVariable Long id) {
    User user = userService.getById(id);
    return Result.success(user);
  }
}

🛡️ 4.9 全局异常处理

import com.example.demo.common.Result;
import org.springframework.web.bind.MethodArgumentNotValidException;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.RestControllerAdvice;

@RestControllerAdvice
public class GlobalExceptionHandler {

  // 处理参数验证异常
  @ExceptionHandler(MethodArgumentNotValidException.class)
  public Result<Void> handleValidException(MethodArgumentNotValidException e) {
    String msg = e.getBindingResult().getAllErrors().get(0).getDefaultMessage();
    return Result.error(400, msg);
  }

  // 处理业务异常
  @ExceptionHandler(RuntimeException.class)
  public Result<Void> handleRuntimeException(RuntimeException e) {
    return Result.error(500, e.getMessage());
  }

  // 处理未知异常
  @ExceptionHandler(Exception.class)
  public Result<Void> handleException(Exception e) {
    e.printStackTrace();
    return Result.error(500, "系统异常，请稍后重试");
  }
}

🔧 4.11 MyBatis Plus核心功能

功能 代码示例 说明
条件构造器 wrapper.eq(User::getUsername, name) 类型安全，避免硬编码字段名
分页插件 new Page<>(pageNum, pageSize) 自动处理分页SQL
逻辑删除 @TableLogic 删除时更新deleted字段
自动填充 @TableField(fill = FieldFill.INSERT) 自动填充创建时间等字段
主键策略 @TableId(type = IdType.AUTO) 主键自增/雪花算法等
链式调用 wrapper.like(...).eq(...).orderByDesc(...) 灵活构建复杂查询

💡 Java后端开发技巧

1. 统一响应格式：所有接口返回Result<T>，前端统一处理
2. DTO与Entity分离：避免暴露数据库结构，增强安全性
3. 参数验证：使用@Validated+注解，自动验证请求参数
4. 逻辑删除：数据不真正删除，便于恢复和审计
5. 事务管理：写操作加@Transactional，保证数据一致性
6. 全局异常处理：统一捕获异常，返回友好错误信息
7. 多环境配置：使用application-{profile}.yml管理不同环境配置

⚠️ 常见错误

1. 忘记配置分页插件：MyBatis Plus分页需要配置PaginationInnerInterceptor
2. 直接返回Entity：应该转换为VO/DTO，避免暴露敏感字段
3. SQL注入风险：使用条件构造器而非字符串拼接SQL
4. N+1查询问题：避免在循环中查询数据库，使用批量查询

🛡️ 费曼图解：MyBatis Plus 为什么能"自动"写SQL？

很多新手困惑："为什么我继承了 ServiceImpl，就能直接调 this.getById() 而不写任何SQL？"

🍔 麦当劳厨房比喻

ServiceImpl = 麦当劳的标准化操作手册
你知道炸薯条需要：放油→加热→放薯条→3分钟→捞出
但你不需要每次炸薯条时都重新写一遍这5步

同样地：
ServiceImpl 就知道增删改查的标准操作：
  • getById(id) = SELECT * FROM table WHERE id = ?
  • save(entity) = INSERT INTO table VALUES (...)
  • updateById(entity) = UPDATE table SET ... WHERE id = ?
  • removeById(id) = DELETE FROM table WHERE id = ?

你只需要告诉它"我要炸薯条"（this.getById(1)），它自己完成剩下的步骤。 这就是"约定优于配置"的本质。

🔧 必须手动配置的：分页插件

但是！分页功能不是"标准操作"——每个数据库的方言不同（MySQL用 LIMIT，Oracle用 ROWNUM）。所以你需要手动告诉MyBatis Plus你用的是什么数据库：

// MyBatisPlusConfig.java - 分页插件配置（必须！）
@Configuration
public class MyBatisPlusConfig {

    /**
     * MyBatis Plus 拦截器配置
     */
    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
        
        // 添加分页插件（指定数据库类型）
        PaginationInnerInterceptor paginationInterceptor = new PaginationInnerInterceptor();
        paginationInterceptor.setDbType(DbType.MYSQL);  // 指定MySQL
        paginationInterceptor.setMaxLimit(500L);         // 单页最大500条，防止一次查太多
        paginationInterceptor.setOverflow(true);         // 超出总页数后回到第一页
        
        interceptor.addInnerInterceptor(paginationInterceptor);
        return interceptor;
    }
}

// 然后在Service中使用分页
public PageResult<User> getUserPage(UserQueryDTO query) {
    // 创建分页对象：第几页、每页多少条
    Page<User> page = new Page<>(query.getPageNum(), query.getPageSize());
    
    // 条件构造器
    LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
    wrapper.like(StringUtils.hasText(query.getKeyword()), User::getUsername, query.getKeyword());
    wrapper.eq(query.getStatus() != null, User::getStatus, query.getStatus());
    wrapper.orderByDesc(User::getCreateTime);
    
    // 执行分页查询
    Page<User> result = this.page(page, wrapper);
    
    return new PageResult<>(result.getRecords(), result.getTotal());
}

⚡ 4.15 Spring Cache + Redis 缓存实战

使用Spring Cache注解 + Redis实现方法级缓存，减少数据库查询压力。

依赖与配置

<!-- pom.xml --> <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-cache</artifactId> </dependency> <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-data-redis</artifactId> </dependency> // CacheConfig.java @Configuration @EnableCaching public class CacheConfig { @Bean public RedisCacheConfiguration cacheConfig() { return RedisCacheConfiguration.defaultCacheConfig() .entryTtl(Duration.ofMinutes(30)) // 默认过期30分钟 .serializeKeysWith(RedisSerializationContext.SerializationPair .fromSerializer(new StringRedisSerializer())) .serializeValuesWith(RedisSerializationContext.SerializationPair .fromSerializer(new GenericJackson2JsonRedisSerializer())) .disableCachingNullValues(); } } // Service层使用缓存 @Service public class UserService { @Cacheable(value = "user", key = "#id") // 查询时自动缓存 public UserVO getById(Long id) { return userMapper.selectById(id).toVO(); } @Cacheable(value = "userList", key = "#pageNum + '_' + #pageSize") public PageResult<UserVO> list(int pageNum, int pageSize) { return userMapper.selectPage(pageNum, pageSize); } @CacheEvict(value = "userList", allEntries = true) // 写入时清除列表缓存 @CachePut(value = "user", key = "#result.id") // 更新返回结果缓存 public UserVO create(UserDTO dto) { ... } @CacheEvict(value = "user", key = "#id") // 删除时清除单条缓存 public void delete(Long id) { ... } @Caching(evict = { @CacheEvict(value = "user", key = "#id"), @CacheEvict(value = "userList", allEntries = true) }) @CachePut(value = "user", key = "#result.id") public UserVO update(Long id, UserDTO dto) { ... } }

多级缓存策略

注解说明使用场景
@Cacheable查询前先查缓存，命中则直接返回查询接口
@CachePut始终执行方法，结果写入缓存新增、更新接口
@CacheEvict删除缓存删除、列表刷新
@Caching组合多个缓存操作复杂更新场景

🚦 4.16 接口限流（Rate Limiting）

防止接口被恶意刷取或突发流量打垮系统。

// 方案1：Guava RateLimiter（单机限流） import com.google.common.util.concurrent.RateLimiter; @Component public class RateLimitInterceptor implements HandlerInterceptor { private final RateLimiter limiter = RateLimiter.create(100.0); // 每秒100个请求 @Override public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) { if (!limiter.tryAcquire(1, TimeUnit.SECONDS)) { response.setStatus(429); response.getWriter().write("{\"code\":429,\"msg\":\"请求过于频繁\"}"); return false; } return true; } } // 方案2：Redis + Lua脚本（分布式限流） @Component public class RedisRateLimiter { @Autowired private RedisTemplate<String, String> redisTemplate; private static final String LUA_SCRIPT = """ local key = KEYS[1] local limit = tonumber(ARGV[1]) local window = tonumber(ARGV[2]) local current = redis.call('INCR', key) if current == 1 then redis.call('EXPIRE', key, window) end if current > limit then return 0 end return 1 """; public boolean isAllowed(String userId, int limit, int windowSeconds) { String key = "rate_limit:" + userId; Long result = redisTemplate.execute( new DefaultRedisScript<>(LUA_SCRIPT, Long.class), List.of(key), String.valueOf(limit), String.valueOf(windowSeconds) ); return result != null && result == 1; } } // 方案3：Bucket4j令牌桶（分布式） // 引入 bucket4j-redis 依赖即可使用令牌桶算法

🔗 4.20 Spring WebSocket实时通信

WebSocket实现服务端主动向客户端推送消息，如通知推送、数据实时刷新。

<!-- pom.xml --> <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-websocket</artifactId> </dependency> // WebSocketConfig.java @Configuration @EnableWebSocket public class WebSocketConfig implements WebSocketConfigurer { @Override public void registerWebSocketHandlers(WebSocketHandlerRegistry registry) { registry.addHandler(myHandler(), "/ws/notify") .setAllowedOrigins("*"); } @Bean public MyWebSocketHandler myHandler() { return new MyWebSocketHandler(); } } // MyWebSocketHandler.java @Component public class MyWebSocketHandler extends TextWebSocketHandler { private static final Map<Long, WebSocketSession> sessions = new ConcurrentHashMap<>(); @Override public void afterConnectionEstablished(WebSocketSession session) { Long userId = getUserIdFromSession(session); sessions.put(userId, session); } @Override public void afterConnectionClosed(WebSocketSession session, CloseStatus status) { Long userId = getUserIdFromSession(session); sessions.remove(userId); } // 推送消息给指定用户 public void sendMessage(Long userId, String message) throws IOException { WebSocketSession session = sessions.get(userId); if (session != null && session.isOpen()) { session.sendMessage(new TextMessage(message)); } } // 广播消息给所有用户 public void broadcast(String message) { sessions.values().forEach(session -> { if (session.isOpen()) { try { session.sendMessage(new TextMessage(message)); } catch (IOException e) { sessions.remove(getUserIdFromSession(session)); } } }); } } // 业务层调用推送 @Service public class NotificationService { @Autowired private MyWebSocketHandler wsHandler; public void notifyUser(Long userId, String content) throws IOException { wsHandler.sendMessage(userId, JSON.toJSONString(Map.of("type", "通知", "content", content))); } }

🧠 费曼自测：你能解释给同学听吗？

Q1：Controller、Service、Mapper各自的职责是什么？

A：Controller = 接电话的客服（只负责接收和返回，不干实质活），Service = 车间工人（真正执行业务逻辑），Mapper = 仓库搬运工（只负责存取数据）。记住口诀：接电→干活→存取。

Q2：为什么要用DTO和Entity分开？

A：就像你去银行不会把保险柜密码贴在大厅墙上。Entity包含数据库所有字段（包括密码hash等敏感信息），DTO只暴露需要给前端的数据。DIFFERENT AUDIENCES, DIFFERENT DATA。

Q3：@Transactional 到底做了什么？

A：就像"一键撤销"按钮。一个操作涉及多个步骤（比如转账：扣钱 + 加钱），如果中间一步失败，前面的所有操作全部撤回。没有它，你可能会"钱扣了但没转过去"。

🧠 记忆口诀：Spring Boot三层心法

C → S → M

Controller接收 → Service处理 → Mapper存取
任何CRUD后端代码都逃不出这三个字母。
