第三章：Vue3 CRUD开发实战 — 扔掉杂物间，拥抱工具箱

费曼学习法：Vue3相比Vue2的改进，就像从"把所有杂物堆在一个大房间"变成"给每个物品一个独立工具箱"。

本章用对比+类比的方式，让你直观理解Vue3为什么更优雅，并通过实战代码快速掌握Composition API、<script setup>、TypeScript、Pinia等现代武器。

本章目标：告别混乱的Options API，用Composition API写出清晰、可复用、类型安全的代码。

🏠 费曼类比：Vue2 vs Vue3 = 老房子 vs 新公寓

Vue2 (Options API) 就像一栋老房子的布局：所有东西按"类型"放——data中的数据放卧室、methods中的函数放客厅、computed放厨房。看起来有规律，但当你需要修一个水管时，你要在卧室（data）、客厅（methods）、厨房（computed）之间来回跑。

Vue3 (Composition API) 就像一个现代公寓：每个功能有独立的工具箱（Hook）。修水管？打开"水管工具箱"，里面管线钳、密封胶、扳手全在一起。不需要满屋子乱找。

这就是为什么大型项目中Vue3更清晰——按功能组织，而不是按类型组织。

🔧 3.3 核心逻辑Hook封装

使用Composition API将CRUD逻辑封装为可复用的Hook函数，实现逻辑与UI分离。

// hooks/useUserCrud.ts
import { ref, reactive, computed } from 'vue'
import { getUserList, createUser, updateUser, deleteUser } from '@/api/user'
import type { User, UserQuery, UserForm } from '@/types/user'
import { ElMessage, ElMessageBox } from 'element-plus'

export function useUserCrud() {
  const loading = ref(false)
  const submitLoading = ref(false)
  const userList = ref<User[]>([])
  const total = ref(0)
  const dialogVisible = ref(false)
  const isEdit = ref(false)

  const queryParams = reactive<UserQuery>({
    pageNum: 1,
    pageSize: 10,
    username: '',
    status: undefined
  })

  const userForm = reactive<UserForm>({
    username: '',
    email: '',
    phone: '',
    status: 1
  })

  const dialogTitle = computed(() => isEdit.value ? '编辑用户' : '新增用户')

  // 查询数据
  const fetchData = async () => {
    loading.value = true
    try {
      const res = await getUserList(queryParams)
      userList.value = res.data.list
      total.value = res.data.total
    } finally {
      loading.value = false
    }
  }

  // 搜索
  const handleSearch = () => {
    queryParams.pageNum = 1
    fetchData()
  }

  // 重置
  const handleReset = () => {
    Object.assign(queryParams, { pageNum: 1, pageSize: 10, username: '', status: undefined })
    fetchData()
  }

  // 新增
  const handleAdd = () => {
    isEdit.value = false
    Object.assign(userForm, { id: undefined, username: '', email: '', phone: '', status: 1 })
    dialogVisible.value = true
  }

  // 编辑
  const handleEdit = (row: User) => {
    isEdit.value = true
    Object.assign(userForm, row)
    dialogVisible.value = true
  }

  // 提交表单
  const handleSubmit = async (formRef: any) => {
    if (!formRef) return
    const valid = await formRef.validate().catch(() => false)
    if (!valid) return

    submitLoading.value = true
    try {
      if (isEdit.value) {
        await updateUser(userForm.id!, userForm)
        ElMessage.success('更新成功')
      } else {
        await createUser(userForm)
        ElMessage.success('新增成功')
      }
      dialogVisible.value = false
      fetchData()
    } finally {
      submitLoading.value = false
    }
  }

  // 删除
  const handleDelete = async (row: User) => {
    await ElMessageBox.confirm(`确认删除用户"${row.username}"吗？`, '提示', { type: 'warning' })
    await deleteUser(row.id)
    ElMessage.success('删除成功')
    fetchData()
  }

  return {
    loading, submitLoading, userList, total,
    dialogVisible, dialogTitle, queryParams,
    userForm, fetchData, handleSearch, handleReset,
    handleAdd, handleEdit, handleSubmit, handleDelete
  }
}

🗃️ 3.6 Pinia状态管理（可选）

对于复杂应用，可以使用Pinia管理全局状态。简单CRUD场景下，Hook封装已经足够。

// stores/userStore.ts
import { defineStore } from 'pinia'
import { getUserList, createUser, updateUser, deleteUser } from '@/api/user'
import type { User, UserQuery, UserForm } from '@/types/user'

export const useUserStore = defineStore('user', {
  state: () => ({
    userList: [] as User[],
    total: 0,
    loading: false,
    queryParams: {
      pageNum: 1,
      pageSize: 10,
      username: '',
      status: undefined as 0 | 1 | undefined
    }
  }),

  actions: {
    async fetchUsers() {
      this.loading = true
      try {
        const res = await getUserList(this.queryParams)
        this.userList = res.data.list
        this.total = res.data.total
      } finally {
        this.loading = false
      }
    },

    async addUser(data: UserForm) {
      await createUser(data)
      await this.fetchUsers()
    },

    async modifyUser(id: number, data: UserForm) {
      await updateUser(id, data)
      await this.fetchUsers()
    },

    async removeUser(id: number) {
      await deleteUser(id)
      await this.fetchUsers()
    }
  }
})

// hooks/useTable.ts - 通用表格分页Hook import { ref, reactive } from 'vue' export function useTable(fetchFn) { const loading = ref(false) const data = ref([]) const total = ref(0) const queryParams = reactive({ pageNum: 1, pageSize: 10 }) const fetchData = async () => { loading.value = true try { const res = await fetchFn(queryParams) data.value = res.data.list total.value = res.data.total } finally { loading.value = false } } const handleSearch = () => { queryParams.pageNum = 1; fetchData() } const handleReset = (defaults = {}) => { Object.assign(queryParams, { pageNum: 1, pageSize: 10 }, defaults) fetchData() } return { loading, data, total, queryParams, fetchData, handleSearch, handleReset } } // 使用示例: const { loading, data, total, fetchData } = useTable(getUserList)

🧠 费曼自测：你能解释给同学听吗？

Q1：Vue3把一个功能的所有代码写在一起，有什么好处？

A：就像工具箱——修水管时不用去三个房间找工具。一个Hook里面 data + methods + computed 全在一起，删功能时也不用到分散的位置删除代码。

Q2：ref() 和 reactive() 什么时候用哪个？

A：ref 像一个小盒子，装单个值（字符串、数字）；reactive 像一个大箱子，装一组相关的值（对象）。简单规则：基本类型用ref，拿不准——统一用ref。

Q3：Pinia 比 Vuex 好在哪里？

A：不需要mutations、完整TypeScript推导、模块自动分离（不需要嵌套在modules里）。
