第二章：Vue2 CRUD开发实战 — 像搭积木一样搭建后台系统

费曼学习法：如果你不能向一个刚学编程的朋友解释清楚Vue是怎么工作的，说明你还没真正掌握。

本章用最直观的代码+最生动的类比，带你从零搭建一个完整的用户管理系统。你会惊奇地发现：原来所有后台页面都是同一套模板的变体。

本章目标：学完之后，你能在1小时内徒手写出任何一个CRUD页面。

🏗️ 费曼类比：Vue页面 = 乐高城堡

搭建一个CRUD页面，就像用乐高搭一个城堡：

地基（API层） = 运输通道，负责从"仓库"（数据库）搬运"积木"（数据）
骨架（组件） = 城堡的结构框架，决定每个房间（页面区域）长什么样
管道（Axios请求） = 传送带，把积木从仓库运到城堡
开关（按钮/事件） = 触发机关，"点击新增" → 打开一扇门 → 放入新积木

一旦你掌握了这四个部分，任何CRUD页面都只是换一套积木颜色而已。

🌐 2.2 Axios请求封装

统一封装Axios，处理请求拦截、响应拦截、错误处理等通用逻辑。

// utils/request.js import axios from 'axios' import { Message } from 'element-ui' // 创建axios实例 const service = axios.create({ baseURL: process.env.VUE_APP_BASE_API, // API基础地址 timeout: 10000 // 请求超时时间 }) // 请求拦截器 service.interceptors.request.use( config => { // 添加Token const token = localStorage.getItem('token') if (token) { config.headers['Authorization'] = `Bearer ${token}` } // 防重复提交：POST/PUT请求添加时间戳 if (['post', 'put'].includes(config.method?.toLowerCase())) { config.headers['X-Request-Timestamp'] = Date.now() } return config }, error => { return Promise.reject(error) } ) // 响应拦截器 service.interceptors.response.use( response => { const res = response.data // 根据后端约定的code判断 if (res.code !== 200) { Message.error(res.msg || '请求失败') // Token过期，跳转登录页 if (res.code === 401) { localStorage.removeItem('token') window.location.href = '/login' } return Promise.reject(new Error(res.msg)) } return res }, error => { let message = '网络异常' if (error.response) { switch (error.response.status) { case 400: message = '请求参数错误'; break case 401: message = '未授权，请重新登录'; break case 403: message = '拒绝访问'; break case 404: message = '请求地址不存在'; break case 500: message = '服务器内部错误'; break case 502: message = '网关错误'; break case 503: message = '服务不可用'; break case 504: message = '网关超时'; break default: message = `连接错误${error.response.status}` } } else if (error.code === 'ECONNABORTED') { message = '请求超时，请稍后重试' } Message.error(message) return Promise.reject(error) } ) export default service

📡 2.3 API接口封装

将API接口统一管理，按模块分类，便于维护和复用。

// api/user.js import request from '@/utils/request' // 查询用户列表（分页+条件筛选） export function getUserList(params) { return request({ url: '/api/users', method: 'get', params }) } // 新增用户 export function createUser(data) { return request({ url: '/api/users', method: 'post', data }) } // 更新用户 export function updateUser(id, data) { return request({ url: `/api/users/${id}`, method: 'put', data }) } // 删除用户 export function deleteUser(id) { return request({ url: `/api/users/${id}`, method: 'delete' }) } // 批量删除用户 export function batchDeleteUsers(ids) { return request({ url: '/api/users/batch', method: 'delete', data: { ids } }) } // 获取用户详情 export function getUserDetail(id) { return request({ url: `/api/users/${id}`, method: 'get' }) } // 更新用户状态（启用/禁用） export function updateUserStatus(id, status) { return request({ url: `/api/users/${id}/status`, method: 'patch', data: { status } }) } // 重置用户密码 export function resetUserPassword(id) { return request({ url: `/api/users/${id}/reset-password`, method: 'post' }) } // 导入用户（Excel） export function importUsers(file) { const formData = new FormData() formData.append('file', file) return request({ url: '/api/users/import', method: 'post', data: formData, headers: { 'Content-Type': 'multipart/form-data' } }) } // 导出用户列表 export function exportUsers(params) { return request({ url: '/api/users/export', method: 'get', params, responseType: 'blob' }) }

🎯 2.5 Vue2 CRUD核心知识点

知识点 说明 使用场景
data() 函数 返回组件的响应式数据 定义表单数据、列表数据、loading状态等
methods 定义组件方法 CRUD操作函数、事件处理函数
computed 计算属性，缓存依赖结果 对话框标题动态切换
created 生命周期 组件创建时调用 初始化加载数据
slot-scope 作用域插槽，获取行数据 表格列自定义渲染、操作按钮
$refs 访问子组件或DOM元素 表单验证、重置表单
async/await 异步操作同步化 API请求处理、简化Promise链

💡 Vue2开发技巧

1. 深拷贝表单数据：编辑时使用 {...row} 避免直接修改表格数据

3. 搜索后重置页码：搜索时 pageNum=1，避免页码超出范围
4. 删除确认：使用 $confirm 二次确认，防止误操作
5. 错误处理：使用 try...finally 确保loading状态正确重置
6. 动态表单：使用 v-for 渲染表单项，注意动态 prop 的写法 'items.' + index + '.field'
7. $nextTick使用：在数据变化后需要操作DOM时使用，如对话框打开后清除验证

⚠️ 常见错误

1. 忘记重置表单：关闭对话框后表单数据未清空，使用 this.$refs.form.resetFields()
2. 响应式失效：直接通过索引修改数组或添加对象属性，应使用 this.$set
3. 内存泄漏：组件销毁时未取消未完成的请求，在 beforeDestroy 中处理
4. 表单验证失效：rules的prop与v-model绑定的字段不一致
5. 表格数据不更新：直接修改数组引用，应使用新数组替换

🧠 费曼自测：你能解释给同学听吗？

Q1：一个CRUD页面由哪四层组成？

A：API层（接口定义）→ Axios请求层（数据传输）→ 组件层（表格/表单/对话框）→ 事件处理层（按钮操作）。每一层各司其职，就像快递包裹从仓库（数据库）经过分拣中心（API）、快递员（Axios）、最终送到你家（页面）。

Q2：为什么搜索后要把页码设为1？

A：想象你在看一本书——当你换了搜索词，书的厚度变了，页码还是原来的那页可能已经不存在了。同理，新搜索条件可能导致总页数减少，保留旧页码可能返回空数据。

Q3：table和form的核心绑定模式分别是什么？

A：table = :data + slot-scope(获取行数据) + @事件(响应用户操作)。form = :model(数据绑定) + :rules(规则) + @validate(提交验证)。记住：表格看数据，表单收发数据。

🧠 记忆口诀：页面制造四步曲

搜 → 列 → 弹 → 存

搜索（Search Form）→ 列表（Data Table）→ 弹窗（Dialog Form）→ 存储（API调用）
任何CRUD页面都逃不出这四个步骤的排列组合。
