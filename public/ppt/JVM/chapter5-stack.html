第五章：栈区详解 — 翻译官的私人草稿纸

栈是JVM中最简单但最容易被误解的区域。这一章用费曼法让你彻底理解：方法调用时发生了什么？递归为什么会导致溢出？局部变量到底存在哪？

📝 5.1 费曼入门：栈是什么？—— 草稿纸比喻

📝 草稿纸比喻

每个翻译官（线程）上班时领一张空白草稿纸（栈）。

每翻译一个章节（调用一个方法），就在草稿纸上划一个框（栈帧），在里面记录：
• 这部分内容在哪（返回地址）
• 需要用到哪些工具（局部变量）
• 中间计算结果（操作数栈）

这章翻译完了，这个框就被划掉（栈帧出栈），空间回收给下一章用。

这就是栈的核心机制：后进先出（LIFO）——最后划的框最先清理。

💥 5.4 StackOverflowError：草稿纸写满了

📝 草稿纸比喻

每个翻译官的草稿纸大小是固定的（默认1MB）。如果任务太复杂——比如无限递归——每个子任务都要划新框，草稿纸很快写满了，"框"划不下了——StackOverflowError！

// 无限递归 - 经典StackOverflow场景
public void recursion() {
    recursion();  // 疯狂调用自己，草稿纸秒满
}

// 结果：Exception in thread "main" java.lang.StackOverflowError

🧠 口诀

栈溢出 = 草稿纸写满了

递归太深 → 栈帧太多 → 栈空间不够 → StackOverflow

⚡ 5.5 栈上的数据为什么快？

栈上数据不需要GC！方法执行完，栈帧自动弹出，空间立即回收。没有垃圾回收的开销，所以栈的分配和释放是纳秒级的。

🎯 逃逸分析优化

JVM会对对象进行逃逸分析：如果发现一个对象不会逃出方法（只在栈上使用，不被返回或传递到外部），就把它分配在栈上而不是堆上。这样方法结束时对象自动销毁，零GC开销！

🎓 本章自检

1. 栈是线程私有的草稿纸，每个方法调用一个栈帧
2. 栈帧四件套：局部变量表 + 操作数栈 + 动态链接 + 返回地址
3. 栈后进先出：最后压入的栈帧最先弹出
4. StackOverflow = 草稿纸写满了（递归太深）
5. 栈上数据不需要GC，方法结束自动回收
