第三章：JVM内存模型 — 翻译官的办公桌布局

前两章我们理解了JVM是翻译公司。这一章走进翻译官的办公室，看看每一件家具放什么、怎么用。
学完你就能解释：new一个对象到底放在哪里？方法执行时变量存在哪？

📊 3.1 办公室布局：五大区域一览

🏢 办公室比喻

堆 = 文件柜：放所有文档（对象），全公司共用
方法区 = 公告墙：贴着公司规章制度（类信息、静态变量），全公司共用
栈 = 私人草稿纸：每人一张，记当前工作内容，用完扔掉
程序计数器 = 书签：标记"我读到第几行了"，每人一个
本地方法栈 = 外线电话：打电话给外部系统（操作系统底层功能）

🥚 3.2 费曼追踪：new一个对象，JVM做了什么？

当你写下 User user = new User()，看起来一行代码，JVM幕后做了6件事：

sequenceDiagram
    participant Code as 👨‍💻 new User()
    participant Method as 📚 方法区
    participant Heap as 📦 堆
    participant Stack as 📝 栈
    
    Code->>Method: 1. 检查User类是否已加载
    Method-->>Code: ✅ 已加载（类信息在方法区）
    Code->>Heap: 2. 分配内存空间
    Heap-->>Code: 返回内存地址 0x1234
    Code->>Code: 3. 初始化零值（字段默认值）
    Code->>Code: 4. 设置对象头（哈希、GC分代年龄、锁标志）
    Code->>Code: 5. 执行构造方法 <init>
    Code->>Stack: 6. 把引用(0x1234)放入栈的局部变量表
    Note over Heap,Stack: ✅ 栈里存引用 → 指向 → 堆里存对象

🥚 孵小鸡比喻

new User() 就像孵小鸡：
1. 先确认这个"物种"存不存在（检查User类是否加载）
2. 在"鸡窝"（堆）里划一块地（分配内存）
3. 铺稻草、放水盆（初始化默认值）
4. 戴脚环标记（设置对象头）
5. 小鸡破壳（执行构造方法！）
6. 在你的"记事本"（栈）上写上"小鸡在X鸡窝"

🧠 口诀

检-分-零-头-构-引

检查 → 分配 → 零值 → 对象头 → 构造方法 → 引用入栈

📐 3.3 对象在堆里长什么样？—— 三件套

一个对象在堆内存中分为三部分：

部分大小包含什么类比
对象头 Header12-16字节Mark Word(哈希、GC年龄、锁标志) + Klass Pointer(类指针)📋 身份证
实例数据 Instance Data取决于字段成员变量（int=4B, long=8B, 引用=4/8B）📄 档案内容
对齐填充 Padding0-7字节补成8的倍数（HotSpot要求）📎 凑整用

关键洞察：对象头里的 Mark Word 是一个"动态身份牌"。同一时刻它可以是哈希码、GC分代年龄、偏向锁线程ID、轻量级锁指针、重量级锁指针。具体是什么——取决于对象当前处于什么状态。

🎯 面试常考

Object obj = new Object() 占多少内存？
对象头(12B) + 实例数据(0B，Object无字段) + 对齐(4B) = 16字节
但开启了压缩指针（-XX:+UseCompressedOops，JDK默认），对象头可能是12字节。

📏 3.4 补充知识：直接内存（堆外内存）

除了五大区域，JVM还能使用堆外内存（Direct Memory）。

📦 快递比喻

文件柜（堆）放文件很方便，但如果你要传50G的大文件给另一个部门，从文件柜里搬来搬去太慢了。

直接内存就像在办公室门口放一个快递柜——东西不经过文件柜，直接交给对方。
NIO的ByteBuffer.allocateDirect()就用这种方式，避免数据在JVM堆和操作系统之间拷贝两次。

⚠️ 注意

直接内存不归GC管，需要手动释放。如果忘了释放，会导致堆外内存泄漏——JVM堆还有空间，但物理内存被吃光了。

🎓 本章自检

1. JVM内存分五大区域：堆（文件柜）、方法区（公告墙）、栈（草稿纸）、PC（书签）、本地方法栈（电话）
2. new对象六步走：检→分→零→头→构→引
3. 对象内存布局三件套：对象头（身份证）+ 实例数据（档案）+ 对齐填充（凑整）
4. 直接内存是"快递柜"——绕过文件柜直接交付，快但不归GC管
